三类新角色定义

🤖

Copilot Agent 副驾智能体

作为检视的"预审员"

职责范围

  • 自动检查格式完整性
  • 术语一致性验证
  • 依赖冲突检测
  • 与历史架构的冲突分析
典型输出： 格式化文档 + 基础缺失清单
⚔️

Adversarial Agent 对抗智能体

扮演"挑战者"角色

职责范围

  • 安全角度攻击设计
  • 性能瓶颈识别
  • 可测试性评估
  • 生成风险报告
典型问题： "如果这个服务挂了，你的降级方案是什么？"
👤

Human Architect 人类架构师

从"通读全文"变为"做决策"

职责范围

  • 审阅Agent生成的摘要
  • 评估关键决策点
  • 处理风险报告
  • 判断业务隐式约束
核心价值： 处理Agent无法应对的非结构化决策

代码与文档一致性检视

文档-代码漂移检测Agent

差异报告包含：
  • Agent认为高风险但不确定的 3个关键点
  • 两个不同Agent产生的 矛盾结论（如安全Agent说风险高，性能Agent说无影响）
  • 涉及资金、用户隐私、核心商业逻辑的 最终裁决

🤥 Agent产生幻觉

Agent检视Prompt模板

📋 通用设计文档检视Prompt

适用于GPT-4、Claude 3.5等通用大模型Agent

角色：你是一名严格的软件架构师。

任务：检视以下 [系统名称] 的设计文档。

重点关注：
1. 完整性：非功能需求（性能、安全）是否有具体指标？
2. 一致性：API请求/响应示例是否与数据模型定义一致？
3. 风险：指出任何单点故障、性能瓶颈、过度工程设计。
4. 依赖：外部服务调用是否有超时和重试的明确说明？

输出格式：
- **致命问题**（必须修改）
- **建议优化**（可改可不改）
- **缺失信息**（需要补充）
- **置信度评分**（对本次检视结论的把握度）

🎯 专项检视Prompt模板

🔒 安全检视

重点检查：认证授权、数据加密、输入验证、SQL注入、XSS风险、敏感数据处理

⚡ 性能检视

重点检查：缓存策略、数据库索引、并发处理、资源池化、异步处理

🧪 可测试性检视

重点检查：依赖注入、Mock友好性、测试数据隔离、边界条件覆盖

核心理念转变

你不需要再问 "我的设计有没有问题？"

而应该问Agent：

"我的设计，和你基于它理解的实现，是否一致？"
